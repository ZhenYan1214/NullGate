     (chain 133), with a <3 minute demo video.

     Pitch: "The refund window earns instead of costs. HSP-Flow is the missing middleware between stablecoin payment rails and DeFi yield."

     ---
     Recommended approach: "HSP-Flow"

     End-to-end flow

     1. Merchant deploys EscrowRouter + MockYieldVault once.
     2. Merchant calls EscrowRouter.createSplitRule(id, recipients[], bps[]) once per rule (e.g. 70% merchant / 20% platform / 10% buyer rebate).
     3. At checkout: backend calls EscrowRouter.registerMandate(mandateId, buyer, amount, splitRuleId, refundWindow) then POST /api/v1/merchant/orders to HSP with pay_to = EscrowRouter.address and
     cart_mandate_id matching.
     4. Buyer opens payment_url, signs EIP-3009 in wallet, HSP broadcasts on-chain transfer to EscrowRouter.
     5. HSP fires webhook. Backend verifies HMAC on raw request body (not parsed JSON). On payment-included or payment-successful, backend (as SETTLER_ROLE) calls EscrowRouter.confirmDeposit(mandateId) →
     contract moves the expected amount into MockYieldVault, records depositedAt.
     6. Anytime before depositedAt + refundWindow: buyer can call refund(mandateId) → principal returns to buyer; accrued yield stays in the vault and flows to merchant at next settle (or to a rebate pool).
     7. After the window: anyone calls settle(mandateId) → vault redeems principal + yield → push-based split across the rule's recipients. Gas-capped at 10 recipients. Last recipient absorbs dust.

     PayFi primitives stacked

     1. Per-payment routing (not per-pool) with invoice metadata binding (cart_mandate_id)
     2. Yield-bearing escrow during refund window = capital efficiency
     3. Programmable split (BNPL-style loyalty rebate as a first-class recipient)
     4. Clean vault interface → real Aave/Morpho-style vault drops in as v2

     Tech stack

     - Contracts: Solidity 0.8.24, Foundry (faster iteration than Hardhat), OpenZeppelin (SafeERC20, ReentrancyGuard, AccessControl). MockYieldVault either hand-rolled linear-accrual or ERC-4626 — whichever
     ships faster.
     - App: Next.js 14 (App Router, TypeScript). Single repo, single Vercel deploy.
     - Wallet/RPC: wagmi + viem + RainbowKit. HashKey Chain testnet (chain 133).
     - HSP client: lib/hsp/client.ts — HMAC-SHA256 request signer, ES256K JWT builder (jose lib, fallback @noble/curves/secp256k1), RFC 8785 canonical JSON via canonicalize npm. Must support a
     HSP_MODE=mock|real env switch (see mock shim below).
     - Storage: No database. Mandate state lives on-chain; frontend reads via viem. Optional data/mandates.json for local dev.

     Contract surface (single file: EscrowRouter.sol)

     struct SplitRule { address[] recipients; uint16[] bps; }   // sum == 10000
     enum Status { None, Registered, Confirmed, Refunded, Settled }
     struct Mandate {
       address buyer;
       uint256 amount;          // expected USDC smallest units
       bytes32 splitRuleId;
       uint64  refundWindow;    // seconds
       uint64  depositedAt;
       uint256 shares;          // from MockYieldVault
       Status  status;
     }
     mapping(bytes32 => SplitRule) public splitRules;
     mapping(bytes32 => Mandate)   public mandates;   // key = keccak256(cart_mandate_id)
     IERC20       public immutable USDC;
     IYieldVault  public immutable vault;
     bytes32      public constant SETTLER_ROLE = keccak256("SETTLER_ROLE");

     function createSplitRule(bytes32 id, address[] calldata recipients, uint16[] calldata bps) external;
     function registerMandate(bytes32 id, address buyer, uint256 amount, bytes32 ruleId, uint64 window) external onlyRole(SETTLER_ROLE);
     function confirmDeposit(bytes32 id) external onlyRole(SETTLER_ROLE) nonReentrant;   // deposits amount into vault
     function refund(bytes32 id) external nonReentrant;                                   // buyer only, before window end
     function settle(bytes32 id) external nonReentrant;                                   // anyone, after window end
     function sweepUnattributed(address to) external onlyRole(DEFAULT_ADMIN_ROLE);        // rescues stray USDC

     MockYieldVault: deposit(amount) -> shares, redeem(shares) -> amount+yield, linear yieldPerSecond accrual. Demo-time yield rate set to 1000% APY so yield is visible within a 2-minute demo window. Pre-fund
     reserve address with USDC from faucet so redemption actually pays out the bonus.

     Showstopper guardrails (from plan review — do these correctly first try)

     1. Dual canonicalization: HMAC bodyHash = SHA256(canonicalJSON(body)). JWT cart_hash = SHA256(canonicalJSON(cart_mandate.contents)). These are different canonicalizations on different objects. Lock them
     in with fixture tests against hand-computed hashes before wiring anything.
     2. Webhook raw body: in Next.js App Router, await req.text() BEFORE JSON.parse. HMAC verification uses bytes, not the parsed object. The signature format is X-Signature: t=<unix>,v1=<hex> and the signed
     message is t + "." + rawBody.
     3. Demo timing: trigger confirmDeposit on payment-included (not payment-successful — that's 20–60 min later). Handle duplicate payment-successful webhook idempotently (no-op if already confirmed). Index
     idempotency by request_id (ID5).
     4. ES256K PEM on Vercel: base64-encode the private key into one env var, decode at runtime. Literal newlines in env vars break jose.importPKCS8.
     5. Nonces: crypto.randomUUID() per request. Timestamps within ±300s.
     6. One-time orders only, not reusable — each payment must bind 1:1 to a mandate row.
