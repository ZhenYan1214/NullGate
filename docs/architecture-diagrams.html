System Architecture Overview
All system components and their interactions. The issuer's KYC allowlist lives entirely off-chain; only the Merkle root is published on-chain. Holders self-admit by submitting a ZK proof of group membership. PrivateRWA enforces compliance on every transfer by querying ComplianceGate — no per-transfer proof is required.

Operational Lifecycle — Sequence Diagram
Message flow across all four operational phases. Phase 1 (root publication) executes once per batch of KYC approvals. Phase 2 (self-admission) executes once per holder per token. Phase 3 and 4 (mint and transfer) execute on demand thereafter — with zero per-transaction proof cost.

Privacy Model — What Is Public vs. Private
Information flow from off-chain identity commitments to on-chain state. The ZK proof bridges the two worlds: it proves group membership without revealing which commitment belongs to which address. The admitted-address set is public by necessity (needed for the transfer hook), but it is cryptographically unlinkable to the off-chain identity list.
Private — Issuer Backend Only
Identity Commitment List
[ c₀, c₁, c₂, …, cₙ ]
Each commitment = H(trapdoor, nullifier).
Stored in data/group.json. Never published on-chain. Not in any event log.
Semaphore Merkle Tree
Full tree structure including all leaves.
Issuer backend uses it to produce Merkle paths for holder proof generation. The tree itself is never broadcast.
Identity ↔ Address Mapping
The issuer knows which real-world identity provided which commitment.
This mapping exists only in the issuer's off-chain KYC records.
Total Group Size
How many investors are KYC-approved is not inferrable from on-chain state. Only the root (a single hash) is public.

Public — On-Chain (Anyone Can Read)
Merkle Root
ComplianceGate.currentRoot() → uint256
A 32-byte hash that commits to the full group state.
Cannot be used to enumerate members.
Admitted Address Set
ComplianceGate.admitted[token][addr] → bool
Public by necessity: the transfer hook must check it without a per-tx proof.
Unlinkable to the off-chain commitment list.
Nullifier Hashes (Used)
ComplianceGate.usedNullifier[hash] → bool
Prevents double-admission. The hash is derived from identity internals and the token scope — not reversible to an identity or address.
Token Balances & Transfers
Standard ERC-20 events and balanceOf().
Transaction-level privacy (shielded pools, stealth addresses) is explicitly out of scope for this protocol.
Unknowable — Even to Blockchain Observers
Which commitment corresponds to which admitted address
How many investors are in the off-chain group
Which approved investors have not yet self-admitted
Whether two admitted addresses belong to the same group member
