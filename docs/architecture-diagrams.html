System Architecture Overview
All system components and data flow. The issuer's KYC allowlist is stored off-chain in Upstash Redis (Vercel production) or a local JSON file (development). Only the Merkle root is published on-chain. Holders self-admit via ZK proof. PrivateRWA enforces compliance on every transfer by querying ComplianceGate.

Operational Lifecycle — Sequence Diagram
Message flow across all four phases. Phase 1 runs once per batch. Phase 2 runs once per holder per token. Phase 3–4 run on demand — zero per-transaction proof cost. The Group Store (Upstash Redis on Vercel, local JSON in dev) persists off-chain state.

Privacy Model — What Is Public vs. Private
Data flow from off-chain identity commitments to on-chain state. The ZK proof bridges the two domains without leaking which commitment corresponds to which address. The admitted-address set is public by necessity (transfer hook), but cryptographically unlinkable to the off-chain identity list.
Private — Off-Chain Only
Identity Commitment List
[ c₀, c₁, c₂, ..., cₙ ]
Each c = H(trapdoor, nullifier).
Stored in Upstash Redis (prod) or data/group.json (dev). Never published on-chain. Not in any event log.
Semaphore Merkle Tree
Full tree structure with all leaves. Used by issuer to produce Merkle paths for holder proof generation. The tree is never broadcast.
Identity ↔ Address Mapping
Which real-world identity provided which commitment. Exists only in the issuer's off-chain KYC records.
Total Group Size
Number of KYC-approved investors is not inferrable from on-chain state. Only the root (a single hash) is public.

Public — On-Chain (Anyone Can Read)
Merkle Root
ComplianceGate.currentRoot() → uint256
A 32-byte hash committing to the full group. Cannot be used to enumerate members.
Admitted Address Set
admitted[token][addr] → bool
Public by necessity: transfer hook must check without per-tx proof.
Unlinkable to the off-chain commitment list.
Nullifier Hashes
usedNullifier[hash] → bool
Prevents double-admission. Derived from identity internals + token scope. Not reversible to an identity or address.
Token Balances and Transfers
Standard ERC-20 events and balanceOf().
Transaction-level privacy is explicitly out of scope.
Unknowable — Even to Blockchain Observers
Which commitment corresponds to which admitted address
How many investors are in the off-chain group
Which approved investors have not yet self-admitted
Whether two admitted addresses share the same group member
